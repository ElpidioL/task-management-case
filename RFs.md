
### 1. User Management

* **RF1:** Users can create an account.
* **RF2:** Users can log in.

---

### 2. Task Management

* **RF3:** Users can create, read, update, and delete tasks (CRUD).
* **RF4:** Users can mark tasks as completed or not completed.
* **RF5:** Users can filter tasks by status and category.
* **RF6:** The system must support pagination for task lists.

---

### 3. Category Management

* **RF7:** Users can create, update, delete, and list categories to organize tasks.

---

### 4. Task Sharing

* **RF8:** Users can share tasks with other users and allow them to view shared tasks.

---

### 5. External API

* **RF9:** The system must integrate with an external API and allow interaction with it.

---

### 6. Technology Requirements

* **RF10:** The backend must be developed using **Django REST Framework**.
* **RF11:** The frontend must be developed using **React**.

---

### 7. Testing and DevOps

* **RF12:** Backend logic must be tested with **pytest**.
* **RF13:** Frontend must be tested using **Selenium**.
* **RF14:** The application must run using **Docker** and Docker Compose.
* **RF15:** CI/CD pipeline must be implemented.

---

### 8. (Optional)

* **RF16:** The application can be deployed using cloud services such as **Amazon Web Services** or **Microsoft Azure**.
